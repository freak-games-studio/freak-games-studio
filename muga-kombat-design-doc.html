Game Design v0.1 — брейншторм

Детальный разбор каждой механики Muga Kombat + идеи для развития игры.

Текущее состояние кодовой базы

4 курсораhello, muuum, muga-pixeled, muga-train
10 апгрейдовmanual-click, auto-click, 2×stats, splash, 3×longtap, 2×cursor
5 дейликовклики, монеты, лонг-тап, AIM, сплеши
6 ачивокvoid-click, buy-all-shop, splashes, streak, dailies, all-achievements

Сплеши на экране

Реализовано
Что есть сейчас
  • 8 вариантов изображений (based, huh, love, old, omfg, pause, shit, sunflower)
  • Спавн при клике с шансом splashChance (базовый 10%)
  • Анимация splash-pop: scale 0.4→1.15→0.95→1→уходит вверх за 1.8с
  • Трекинг уникальных вариантов через seenSplashVariants
  • Ачивка за сбор всех 8 вариантов
Механика начисления
  • spawnScreenSplash(x, y) вызывается при каждом клике
  • Случайный вариант из массива SPLASH_VARIANTS
  • Автоудаление через 1800ms по id
  • Шанс увеличивается апгрейдом splash-chance (+5% за уровень, макс 80%)
  • Бонус из дейлика collect-splash (+10% на 10 мин)
💡
Идея — Редкие сплеши: ввести градацию редкости (common / rare / legendary). Rare-сплеши дают +монеты при появлении. Legendary — временный буст всем статам на 30 секунд.
💡
Идея — Размер под combo: во время активного комбо-буста сплеши в 1.5× крупнее и имеют golden-vibe эффект (CSS filter: sepia + hue-rotate).
💡
Идея — Кликабельные сплеши: с небольшим шансом (5%) сплеш оказывается «золотым» и кликабельным — даёт бонусные монеты или ускоряет кулдаун лонг-тапа.

Удержание пальца/мыши на айтеме в магазине. Вариант UX: прогресс-баp заполняется 1.5с → покупка совершается. Это позволяет покупать без случайных тапов на мобильном. Муга

Реализовано
Текущая механика клика

Клик по муге → clickMuga(x,y) → проверка миссклика → подсчёт монет (с учётом double-click chance и combo-буста) → спавн сплеша → прогресс комбо-бара → шанс тригера AIM-Муги. Мульти-тач реализован через useMultiTouch — каждый палец регистрируется независимо.

AIM-Муга (уже есть)
  • Шанс срабатывания: aimMugaChance (базовый 5%)
  • 8 мини-муг появляются рандомно на экране
  • Таймер 8 секунд на клик всех
  • Успех → награда x20 от clickPower × 8
  • Провал → штраф к AIM-шансу (×0.2) на 30 секунд
  • Таймер-бар внизу экрана с danger-состоянием
Комбо-бар (уже есть)
  • Заполняется на +0.12 за каждый клик
  • При 100% → активирует буст ×5 на 4 секунды
  • Буст затухает нелинейно (базовый decay + ускорение)
  • Пассивный сброс -0.15 при 800мс бездействия
  • Не реализован отдельный UI-элемент (надо добавить!)
💡
Идея — Визуализация комбо-бара: сейчас нет явного UI для прогресса комбо. Нужен прогресс-бар под мугой с цветовой градацией: зелёный→жёлтый→красный при 100% взрыв-анимация и временное свечение вокруг муги.
💡
Идея — Муга-реакции: разные спрайты муги в зависимости от состояния — обычный, long-tap, overflow (красный), cooldown (серый), во время AIM-режима (испуганный).
💡
Идея — Звуки: лёгкие звуковые эффекты на клик, лонг-тап, комбо-взрыв, появление AIM-муги. Web Audio API — без внешних файлов через синтез или короткие base64-аудио.

Лонг-тап

Реализовано
Логика механики

Удержание кнопки/тача → useLongTap.start() → setInterval тикает монеты каждые tickIntervalMsмс → по истечении maxDurationMs переполнение (overflow) → stop(true) штраф (монеты не засчитываются) → кулдаун. Правильный лонг-тап: stop(false) → монеты засчитаны, кулдаун.

Текущие статы (формулы)
СтатФормула
Множитель2 + level×0.5 + cursorLevel×0.2
Макс. время3000 + level×1000 мс
Интервал тикаmax(100, 300 − level×50) мс
Кулдаун5000 мс (пока фиксированный)
Монеты/тикceil(clickPower/2 + level×0.5)
Визуальные состояния
  • Обычный: муга без эффектов
  • Активный: scale(1.08) + SVG-кольцо с прогрессом
  • Overflow: красная drop-shadow + статус «!»
  • Cooldown: opacity 0.6 + статус «...»

⚠️ Нет UI для кулдауна (числовой обратный отсчёт). Сейчас только opacity.

💡
Идея — Обратный отсчёт кулдауна: показывать число секунд под мугой во время кулдауна вместо просто «...». Можно анимировать как старый LCD-дисплей с Minecraft-шрифтом.
💡
Идея — Апгрейд кулдауна: сейчас кулдаун захардкожен в 5000мс. Добавить апгрейд long-tap-cooldown (3 уровня, −1с за уровень) в магазин в таб лонг-тапа.
💡
Идея — Хапик («хороший момент»): в случайный момент в последние 500мс перед overflow появляется мерцание/звук — сигнал о «sweet spot» зоне, за отпускание в которой дополнительный бонус +20% монет.
💥

Комбо-клик

Частично
Что реализовано

AIM-Муга — это и есть комбо-клик: муга распадается на 8 мини-муг. Кнопки позиционируются через position: absolute в процентах (8%–92% по x/y) с рандомным размером 48–95px. Анимация bounce (mini-bounce). При клике — сплайс из массива, отсчёт в aimMugaRemaining.

Связь с курсорами — TODO

Задуманная механика: во время комбо-клика размер курсора увеличивается (hitbox шире). Сейчас sizeMultiplier в game-cursor.vue всегда возвращает 1. Нужно передавать состояние aimMuga.active и применять множитель на основе уровня курсора.

Шанс срабатывания через мугу

Текущий aimMugaChance: базовые 5% + бонус от дейлика trigger-aim-muga (+0.5%, 15 мин). Нет апгрейда для увеличения шанса в магазине. Максимум захардкожен 30%. Пенальти после провала сбрасывает шанс до 1% (×0.2 фактор).

💡
Идея — Разные конфигурации AIM-муги: количество мини-муг зависит от уровня апгрейда (8 → 12 → 16). Больше штук = больше reward. Добавить апгрейд aim-muga-count в магазин.
💡
Идея — Разные паттерны движения: мини-муги не просто bouncing, а медленно дрейфуют к краям экрана. Чем выше уровень курсора — тем медленнее двигаются (лёгче кликать).
💡
Идея — Цепной AIM: если завершить AIM-мугу за 3 секунды — немедленно запускается второй AIM без кулдауна, с удвоенной наградой.
⬆️

Апгрейды

Реализовано
Система уровней

Каждый апгрейд — массив UpgradeLevel[]. При покупке buyOrUpgrade(id) увеличивает level в ownedUpgrades. В магазине показывается следующий уровень через nextLevelData. В инвентаре показывается текущий уровень (currentLevel / maxLevel). Все эффекты апгрейдов вычисляются реактивно в computed через getUpgradeLevel(id).

Механика «пункт на улучшение»

При покупке апгрейда в магазине он сразу же появляется в инвентаре с уровнем 1. В магазине продолжают показываться следующие уровни апгрейда как отдельные строки (или та же строка с updated cost). Логика в shop-store.ts: upgradeShopItems всегда показывает nextLevel данные для каждого апгрейда.

💡
Идея — Апгрейд для AIM-шанса: сейчас нет апгрейда повышающего aimMugaChance в магазине. Добавить aim-muga-chance (3 уровня, +5% каждый) в таб апгрейдов или отдельный таб «AIM».
💡
Идея — Апгрейд кулдауна лонг-тапа: long-tap-cooldown (3 уровня: 5с → 4с → 3с → 2с) в таб лонг-тапа магазина.
💡
Идея — Визуализация прогресса в магазине: маленький прогресс-бар рядом с ценой показывает сколько уровней куплено (filled dots или segmented bar). Прокаченный до максимума апгрейд показывает бейдж MAX вместо цены.

Дейлики

Реализовано
Правила системы
  • Максимум 3 активных дейлика
  • При входе в новый день — незавершённые остаются, добавляются новые до 3
  • Дейлик при завершении даёт временный бонус к стату на N минут
  • Бонус отображается пульсирующим бейджем BONUS
  • Каждые 60 секунд бонусы пересчитываются через pushBonuses()

Статы

Реализовано
GameStats — вычисляемый объект

Все статы — computed в game-store.ts. Формула: базовое значение + бонус апгрейда + dailiesBonuses + achievementBonuses. Апгрейды и ачивки хранятся в отдельных ref объектах и суммируются реактивно. Дейлики пересчитываются через watcher каждую минуту.

doubleClickChance0%stats-double-click, ачивки ×3, click-100 дейликmax 50%
autoClickIntervalMs1000мсauto-click апгрейд, complete-10-dailies ачивка, earn-1000 дейликmin 200мс

💡
Идея — Детальная панель статов: в текущей stats-panel.vue показывать не просто итоговое значение, а разбивку: «базовый + апгрейды + дейлики + ачивки = итого» при hover на строку.

Лидерборд

Локальный
Текущая реализация

Локальный localStorage-лидерборд. Отправка очков по имени игрока. Хранит до 50 записей, показывает топ-20. Сортировка по coins desc. Отображает имя, счёт и дату. Форма скрывается после отправки (submitted ref).

💡
Идея — Онлайн-лидерборд: Firebase Realtime Database или Supabase. Простой POST с именем и счётом. Подсветка строки текущего игрока если его запись уже есть. Row с «Ваш результат» sticky внизу таблицы.
💡
Идея — Еженедельный сброс: лидерборд сбрасывается раз в неделю. Хранить hall-of-fame с топ-3 прошлой недели отдельно. Создаёт повторяемое соревновательное событие.
💡
Идея — Категории лидерборда: всего монет / кликов / лонг-тапов / ачивок. Переключение между категориями табами.
